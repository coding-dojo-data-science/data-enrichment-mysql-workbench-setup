SQL Sub-queries

  • Understand what a subquery is
  • Explore examples of subqueries in different places

Subqueries are also known as inner queries or nested queries.  A subquery is used to perform the query in multiple steps.  The subquery is placed in parenthesis and is completed first.  The subquery must be able to run on its own. Subqueries can occur anywhere within the main query.  To help you understand how subqueries are used, we will look at at example of a subquery occurring in three different places:

  • WHERE
  • SELECT 
  • FROM

We will continue to work with the world database for these examples.

USE world;

Subquery in a WHERE clause

The most common place to use a subquery is within a WHERE clause.  

Consider this task: Select all countries where the gnp is greater than the average gnp.  

The code below shows how to achieve this by using a subquery within the WHERE statement.  The subquery will be processed first and will determine the average gnp.  Then that value will be used in the main query when filtering for countries where the gnp is greater than average.  Notice, that it is good practice to indent the subquery. Also note that you could run the subquery alone to get an idea of the results before including it in the main query.

SELECT name, gnp
	FROM countries
	WHERE gnp > 
	  (SELECT AVG(gnp)
	  FROM countries);

Subquery in a SELECT clause

For this example we would like to compare each country's population with the average population of all countries. We want to include the name of the country, its population, and the average population in each row.  Let's limit our results to 5.  This is an example of how a subquery could be used to produce this result. 

SELECT countries.name, countries.population,   
	  (SELECT AVG(countries.population) FROM countries)
	FROM countries
	LIMIT 5;

Subqueries in the SELECT clause can also be used as an alternative to performing a join.  For example, to list the number of languages associated with each country ordered from highest to lowest, you could perform the following:

 SELECT countries.name,
	  (SELECT COUNT(*)
	   FROM languages
	   WHERE countries.id = languages.country_id) AS lang_num
	FROM countries
	ORDER by lang_num DESC;

Subquery in a FROM clause

Next, lets say we want to know the average value of the number of countries per region. This requires taking an aggregate(average) of an aggregate (count). The subquery will first calculate the count of each country per region.  The outer query will then find the average of that result.

Note that we are required to provide an alias for the subquery. In this case, we used "sub".  

SELECT AVG(num_countries)
	FROM
	  (SELECT COUNT(*) AS num_countries
	  FROM countries
	  GROUP BY region) AS sub;


Summary

This lesson introduces the concept of subqueries by providing examples of how a subquery might be used in different clauses of the main query.  It is important to realize that there are usually multiple ways to achieve the output you are seeking, and subqueries are another tool to add to your growing SQL toolbox.
