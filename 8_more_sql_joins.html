To explain join types, it is easiest to demonstrate the different joins with a simple database.

CREATE DATABASE teachers;

Inner Join

We practiced Inner Joins in a previous lesson as they are the most common. Remember, if you just use JOIN without specifying, you will complete an INNER JOIN by default.  

An inner join requires each row in the two joined tables to have matching column values for the ON condition.  If there is a NULL value, the ON condition cannot be matched, and that row is excluded from the output table.  

Try an inner join that selects the last name of the teacher and the name of their subject. Note that we have used aliasing for the table names.

SELECT t.LastName, s.SubjectName 
	FROM teacher AS t
	INNER JOIN subject as s
	ON t.SubjectID = s.subjectID;

 


Notice that the results do NOT include the teacher named Rutland because they have a NULL value for SubjectID.

This may not be the result you wanted. What if you wanted to include all the names of the teachers and their subject,  but also want to include the names of teachers who are not associated with a subject?  

Left Outer Join

The result of a left outer join (left join)  contains all rows of the "left" table (first table), even if their is no matching value in the "right" table (second table).  

For example, a LEFT JOIN  allows us to list all of the teachers and show their subject, as well as include the teachers who do not have a subject.


SELECT t.LastName, s.SubjectName 
	FROM teacher AS t
	LEFT JOIN subject as s
	ON t.SubjectID = s.subjectID;


Notice, that this output includes Rutland even though it has a NULL value for the SubjectName column.

Right Outer Join

A right outer join (right join) is very similar to a left outer join, but it is the opposite. Every row from the "right" table (second table) will appear even if there is  no matching value in the "left" table (first table.) 

In our example, we will see that all of the subjects are listed even if they do not have a corresponding teacher.

SELECT t.LastName, s.SubjectName 
	FROM teacher AS t
	RIGHT JOIN subject as s
	ON t.SubjectID = s.subjectID;

Notice that Science is included as a subject, but since no teachers are assigned to science, the LastName column is NULL.

Right and left joins do the same thing (just in opposite) so you can achieve the same results by switching from right to left and switching the order of the tables.  Try it and confirm that you can get the same results!


Full Outer Joins

Another type of join that is used in many SQL database is a full outer join.  Note, however, that MySQL does not directly support the use of full joins.  In case you use a different tool, or need to conceptually understand a full outer join (sometimes just called FULL JOIN or OUTER JOIN) this is an example of how it might look:

This will NOT work in MySQL!

# This will NOT work in MySQL!
	SELECT t.LastName, s.SubjectName 
	FROM teacher AS t
	FULL OUTER JOIN subject as s
	ON t.SubjectID = s.subjectID;

There is a workaround in MySQL that will yield the results we are seeking without needing the functionality of the FULL OUTER JOIN keyword.  It involves using UNION with a Left and Right Join. 
SELECT teacher.LastName, subject.SubjectName
	FROM teacher 
	LEFT JOIN subject
	ON teacher.SubjectID = subject.SubjectID
	UNION 
	SELECT teacher.LastName, subject.SubjectName
	FROM teacher
	RIGHT JOIN subject ON teacher.SubjectID = subject.SubjectID;

Notice that you have all of the teachers and all of the subjects despite NULL values appearing in each column.


Summary

This lesson demonstrated the difference in results that you would get from applying inner, left, right, and full joins.  When there are missing values in your database, it is especially important to consider what you want your results to include.  MySQL does not have a FULL OUTER JOIN, but it can be achieved by using UNION with a left and right join.  It is helpful to practice all of these joins on a simple database such as the one here so that you can recognize the difference in outputs.
