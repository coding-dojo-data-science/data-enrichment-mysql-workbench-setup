Intro to Joins

Objectives:

  • Identify foreign keys in an ERD
  • Apply INNER JOINS to obtain information from multiple tables
  • Use aliasing with table names

What is a Join?

Joins combine columns from one or more tables into a single output table.

There are several types of joins that often can cause confusion especially when first starting out with SQL queries.  Before exploring the different types of joins, it can be easiest to become familiar with the most common type: INNER JOIN.  In fact, INNER JOIN is so common that simply typing the keyword JOIN will refer to an INNER JOIN.

It can be helpful to learn the applications of joins by working with an existing database and seeing which type of questions require a join.  For this lesson, we will continue to explore the "world" database that you should have loaded into MySQL Workbench in an earlier lesson. 

Foreign Keys

A foreign key provides the link between two tables.  

Take some time to explore the ERD above to determine which columns of data can be used to match one table to another.  Notice that both the the "languages" table and the "cities" table have a column called "country_id" that will match with the "id" column in the countries table.  Also notice how the diamond to the left of the country_id column is a red/pink color.  In MySQL workbench, this indicates that this column serves as a foreign key.  

Join Example 1:

Let's start by exploring a question that will require joining the countries table and the languages table: 

Which countries speak Dutch?

The JOIN keyword is used to indicate which additional table is needed and the ON keyword shows the link between the two tables with an =. This will be involve the foreign key of one table and the matching reference column in the other.

Because more than one table is involved, we use the syntax: table.column.  For example to select the name column from the countries table we write: countries.name  

SELECT countries.name
FROM countries
JOIN languages
ON countries.id = languages.country_id
WHERE languages.language = 'Dutch';

The results above show all the countries where Dutch is included as a language in the database.


JOIN Example 2:

Try another JOIN to answer the following:

List all of the cities in Belgium.

SELECT cities.name
FROM countries
JOIN cities 
ON countries.id = cities.country_id
WHERE countries.name = 'Belgium';

The results above show all of the cities listed in the database for Belgium.


JOIN Example 3:

Try another join to see:

Which languages are spoken in the United States?

SELECT countries.name, languages.language
FROM countries
JOIN languages 
ON countries.id = languages.country_id
WHERE countries.name = "United States";


The output above shows all the languages included in the database for the United States.


Using Aliases with Table Names

Since writing the table.column syntax can be a bit cumbersome, we can shorten our queries by aliasing the table names with AS.  This does NOT change the name of the table. It is just a temporary change to make writing the query more efficient.

It is common to use the first letter or two of the table name as the alias. The code below shows the same query as above, but using aliasing for the table names.

SELECT c.name, l.language
FROM countries AS c
JOIN languages AS l
ON c.id = l.country_id
WHERE c.name = "United States";

Note that aliasing is optional, but it is important that you understand it so that you can read the syntax when others use it.  Also note that any alias can only be used once.  For example, in our database, if we choose to use "c" for "countries" we cannot use it again for "cities." In this case we might choose not to alias cities or to use something like "ci" to shorten it.  Try practicing the first few examples on this page using aliasing to get more familiar with the practice.  


Summary:

This lesson introduced the concept of joins by practicing several inner joins.  In order to make a join, there must be a common column in each table.  A foreign key in one table references the primary key in another.  We specify the table name with the column using table.column.  It is common to alias table names with the first letter or two of the table name.  The nature of databases makes joins an essential SQL command. You will get more comfortable with joins as you continue to practice.
